### **3-5 条推荐研究主线**

#### **1. 瘤旁→肿瘤核心的"稳态丧失 → 应激适应"轴**（超强证据等级）

**聚焦问题**：Homeostatic_MG (100% 外周) → HS-UPR TAM (86% 肿瘤) 之间是否存在连续的状态转换轨迹？

**推进方向**：
- 用 RNA velocity / pseudotime (monocle3/scVelo) 在整合的三个 dataset 上推断 **Homeostatic → Stress 的转变轨迹**
- 关键检验点：过渡态细胞（中间表型）在 Edge 区域是否富集（支持三位置 GBM 的细致空间）
- 转录因子动态：从 ZFP36L2 (stasis) → ATF4/CHOP (UPR) 的激活时序
- **验证方向**：spatial transcriptomics 确认梯度，或 immunofluorescence 验证关键标记物的空间分布

---

#### **2. 补体-清道夫双重激活的 TAM "中间功能态" 定位**（中等证据）

**聚焦问题**：Complement_Scavenger_TAM 何以跨区域位置相对中立？它代表"M2 样清理功能"还是"M1→M2 的过渡"？

**推进方向**：
- **细胞互作假说**：此 program 是否富集在基质-免疫-肿瘤 niche 的交界？用 CellChat/LIANA 测定其与 CAF/内皮/肿瘤细胞的关键通讯
- **代谢特征**：lipid uptake (LDL/HDL 通路) 与 complement-mediated phagocytosis 的共激活是否指示"补体调理吞噬"
- **位置再检视**：snGBM 99% Peri vs scGBM 无强位置偏好，差异源于细胞类型输入（Microglia+Macrophage 混合？）还是样本异质性？
- **直接实验**：FACS 分离此 program 的细胞，测定补体受体(CR1/CR3/CR4) 与脂质代谢标记的共表达

---

#### **3. scGBM 的 Mg_Inflam TAM 与 snGBM 的完全缺失之谜 → Microglia-Macrophage 分工假说**（强差异等级）

**聚焦问题**：为何 scGBM 中 Mg_Inflam 占 35-40%，而 snGBM (纯 Macrophage) 中完全无对应？

**推进方向**：
- **清晰假说**：Mg_Inflam 程序代表 **Microglia 特异的核心功能**；macrophage 无法复制此高动力 IEG (JUN/FOS/KLF2) 激活
- **验证策略**：
  1. 在三个 dataset 中分离 Microglia vs Macrophage 两个族群，分别对这两者做 Mg_Inflam signature 评分 
  2. 假设：Microglia 高评分 & 肿瘤富集；Macrophage 低评分 or 不存在
- **机制推断**：CX3CR1 (Microglia marker) 与 Mg_Inflam 模块表达的相关性强度
- **临床意义**：Microglia 清除可能选择性消灭 Mg_Inflam 功能，保留 Macrophage-driven 清道功能 → CAR-T 联合 CSF1R 抑制的设计依据

---

#### **4. HS-UPR TAM 的肿瘤核心应激微环境征**（超强证据，已高度稳健）

**聚焦问题**：HS-UPR 完全肿瘤限制（snGBM 86% Tumor），如何与 HIF1A/缺氧轴互联？

**推进方向**：
- **整合验证**：在同样的样本上测 HIF1A activity (e.g., signature score) 与 HS-UPR 评分的相关性
- **上游驱动**：缺氧 vs 营养饥饿 vs 肿瘤细胞分泌物，哪个是主驱动？用 bulk/空间数据反卷积计算 pO2 gradient
- **治疗靶点**：
  - 抑制 HSP90/DNAJB1 是否选择性杀伤应激 TAM（相对于稳态MG安全）
  - 缓解 UPR 是否重启 macrophage 的清道功能（从应激模式重编程到清理模式）
- **预测价值**：HS-UPR score 与免疫检查点反应的关联

[三位置 GBM/1_myeloid_subclustering.ipynb](三位置%20GBM/1_myeloid_subclustering.ipynb#L1574-L1582) 的价值最高在空间分辨率。它把 Peripheral、Peritumoral 统一到 Peripheral_like，把 Core、Tumor 统一到 Tumor_like，但同时保留了 Peripheral、Edge、Core 三层趋势。真正该盯的是 [三位置 GBM/write/1_myeloid_subclustering/myeloid_subtype_tissue_trend_summary.csv](三位置%20GBM/write/1_myeloid_subclustering/myeloid_subtype_tissue_trend_summary.csv) 和 [三位置 GBM/write/1_myeloid_subclustering/myeloid_program_l1_tissue_trend_summary.csv](三位置%20GBM/write/1_myeloid_subclustering/myeloid_program_l1_tissue_trend_summary.csv)：HS_UPR、Hypoxia、Scavenger_IS 都往 Core 走，Systemic_Inflam 和 Complement_IS 更偏 Peripheral。这个 notebook 最适合当发现集。

[scGBM/4_Sub_clustering.ipynb](scGBM/4_Sub_clustering.ipynb#L521-L529) 的价值最高在“全景 atlas”。它的 myeloid 输入最宽，包含 Macrophage、Microglia、Monocyte、Neutrophil，所以最容易把程序拆得细，但也最容易把来源差异混进来。当前你刚重跑出来的组织倾向在 [scGBM/write/4_Sub_clustering/myeloid_subtype_by_tissue_position_fraction_within_label.csv](scGBM/write/4_Sub_clustering/myeloid_subtype_by_tissue_position_fraction_within_label.csv) 里已经很清楚：Mg-Inflam TAM 和 Lipid-metabolism TAM 偏 Peripheral_like，而 Homeostatic MG、IFN-resp TAM、Systemic-Inflam TAM 偏 Tumor_like，Complement-Scavenger TAM 和 HS-UPR TAM 则接近中间态。这个 notebook 更适合当参考图谱和统一命名骨架。

**最关键的冲突**
1. Homeostatic MG 这个标签现在不能直接跨 notebook 等同。snGBM 的 Homeostatic MG 是典型稳态 microglia，注释写得很明确，P2RY12、NAV3、SLC1A3，而且 99% Peritumoral，[snGBM/5_snRNA_myeloid_subclustering_v2.ipynb](snGBM/5_snRNA_myeloid_subclustering_v2.ipynb#L552-L552)。但 scGBM 的“Homeostatic MG”其实带有明显的 MT1X、MT2A、CSTB 特征，[scGBM/4_Sub_clustering.ipynb](scGBM/4_Sub_clustering.ipynb#L525-L525)，并且在你刚重跑的新结果里是 95.5% Tumor_like，[scGBM/4_Sub_clustering.ipynb](scGBM/4_Sub_clustering.ipynb#L615-L615)。这更像一种 metal-stress microglia，而不是真正的 homeostatic microglia。这个标签漂移如果不先处理，会直接把研究主线带偏。
2. Mg-Inflam TAM 不够稳，至少不能当三本共通主线。它在 scGBM 里是最大群，[scGBM/4_Sub_clustering.ipynb](scGBM/4_Sub_clustering.ipynb#L500-L500)，而 snGBM 并没有一个对应的独立群。这更像是一个依赖 scRNA 输入组成、尤其依赖 microglia 混入程度的程序，而不是“所有 GBM 髓系都共享”的核心状态。
3. Systemic-Inflam 也不够稳。三位置里它更偏 Peripheral，[三位置 GBM/write/1_myeloid_subclustering/myeloid_subtype_tissue_trend_summary.csv](三位置%20GBM/write/1_myeloid_subclustering/myeloid_subtype_tissue_trend_summary.csv)，但当前 scGBM 新结果里它是 Tumor_like，[scGBM/4_Sub_clustering.ipynb](scGBM/4_Sub_clustering.ipynb#L617-L617)，而 snGBM 又没有一个非常干净的一一对应群。这说明它更像平台和谱系敏感程序，不适合作为第一主线。

**我建议的研究主线**
1. 主线一：外围/边界到核心的应激递进。
核心问题不是“哪个 subtype 多了”，而是 Complement_Scavenger/Scavenger_Lipid 这类修复-吞噬程序，如何在 Edge 过渡成 HS-UPR/Hypoxia/Stress_Hypoxia。三位置负责定义梯度，snGBM 负责验证 Tumor vs Peritumoral，scGBM 负责给出全景参考和 label space。
2. 主线二：把“空间轴”和“谱系轴”拆开。
建议明确分成 Program axis 和 Lineage axis 两层。Program axis 看 Complement_Scavenger、Scavenger_Lipid、Stress_HSUPR、Stress_Hypoxia。Lineage axis 看 Microglia-like、Macrophage-like、Monocyte-like。这样就能解释为什么 scGBM 会出现 Mg-Inflam 和“肿瘤型 Homeostatic MG”，而 snGBM 不会。
3. 主线三：把 Complement_Scavenger 定义成边界接口态，而不是终点态。
它在三位置偏 Peripheral，但不是最外围；在 scGBM 接近中间态；在 snGBM 又常常出现在 Peritumoral。它最像边界清除/修复接口，而不是单纯的“好”或“坏” TAM。这个状态非常适合往细胞通讯、血管邻近、基底膜重塑上扩。

**最值得你现在马上做的，不是继续加 subtype，而是先统一分析框架**
1. 先把三本都收敛到 myeloid_program_l1 层，不再直接比较原始 subtype 名字。
2. 把 scGBM 里的 Homeostatic MG 重新审一下命名，至少和 snGBM 的真正稳态 microglia 区分开。
3. 以三位置为发现集，先做一个 program 级别的 weighted center 排序，再用 scGBM 和 snGBM 验证哪些 program 在二位置里也保留同方向。
4. 真正的论文主图建议围绕 Peripheral/Edge/Core 或 Peripheral_like/Tumor_like 的程序迁移，不要围绕 Mg-Inflam 这种单数据集强信号来搭总叙事。
