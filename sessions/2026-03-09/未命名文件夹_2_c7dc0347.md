## 致命问题（会导致错误结果或运行失败）

### 1. **缺少 log2 转换（最关键！）**
-

：`normalizeBetweenArrays` 后 `range(exp)` 为 **0.16 ~ 33697**，远超 20
- 代码注释说"范围在20以内说明已经log转换"，但实际 **并未做 log2 转换**
- 直接拿原始值做 `limma` 差异分析，`logFC` 是在 **线性尺度** 上计算的，完全没有意义
- 对比实测：不做 log2 得到 **21954 个 DEG**（严重膨胀），做 log2 后只有 **3767 个**

### 2. **`idmap2` 包未安装，且安装方式有问题**
-

：`install_github("jmzeng1314/idmap2")` — 这个包可能已不在 GitHub 上维护
- `gitcreds::gitcreds_delete()` — **危险操作**，会删除系统已保存的 Git 凭证
- 每次运行脚本都会尝试安装，不合理
- 你的系统中 **`hgu133plus2.db` 已安装**，可作为替代

### 3. **探针 ID 未成功转换导致下游全链路失败**
- 因为 `idmap2` 加载失败，探针无法转换为基因 Symbol
-

：`exp <- exp %>% inner_join(ids, by = "ID")` — `ids` 对象不存在，会报错
-

：`bitr(diff.genes, fromType = "SYMBOL", ...)` — `diff.genes` 仍然是探针 ID（如 `1007_s_at`），不是 SYMBOL，`bitr()` 会转换失败

### 4. **`ggplot()` 中使用了无效参数 `showCategory`**
-

：`ggplot(go.re.se, showCategory = 20, aes(...))` — `showCategory` 是 `clusterProfiler` 的参数，不是 `ggplot2` 的。会报 `unused argument` 错误
