| 维度 | `sc.pp.scrublet()`（scanpy 封装） | `scrublet.Scrublet()`（原始库） |

| **自动预处理** | 内部自动做 normalize → log1p → HVG → PCA，无需提前处理 | 也有内置预处理，但需手动传参控制（`min_counts`, `min_cells`, `min_gene_variability_pctl`, `n_prin_comps`） |
| **batch 支持** | 有 `batch_key` 参数，可自动按样本分批运行 | 无，需自己写 for 循环按样本拆分 |
| **阈值** | 有 `threshold` 参数，也支持自动阈值 | 同样有自动阈值（双峰检测），可通过 `call_doublets(threshold=)` 覆盖 |

| **输出** | 自动写入 `adata.obs['predicted_doublet']` 和 `adata.obs['doublet_score']` | 返回 `(scores, predictions)` 元组，需手动赋值回 obs |
| **自动预处理** | 内部自动做 normalize → log1p → HVG → PCA，无需提前处理 | 也有内置预处理，但需手动传参控制（`min_counts`, `min_cells`, `min_gene_variability_pctl`, `n_prin_comps`） |
| **batch 支持** | 有 `batch_key` 参数，可自动按样本分批运行 | 无，需自己写 for 循环按样本拆分 |

| **底层** | **内部就是调用 scrublet 库**，只是做了 AnnData 适配封装 | 原始实现 |

### 核心结论

**算法完全一样**——`sc.pp.scrublet()` 底层就是调用 `scrublet.Scrublet`，区别仅在于：

1. **便利性**：scanpy 封装自动处理 AnnData 输入输出，省去手动拆矩阵、写回 obs 的步骤
2. **batch 分组**：scanpy 版有 `batch_key` 参数可一行搞定逐样本运行，原始库需手动循环
3. **参数暴露程度**：原始库可以精细控制更多内部参数（如 `min_counts`, `min_cells`, `min_gene_variability_pctl`, `n_prin_comps`），scanpy 封装只暴露了常用参数
