## 问题诊断（为什么会乱）

- 重复使用 merge(by="Row.names") 会不断创造临时列 Row.names，过程中行顺序被打乱，且 rownames 丢失、再赋值，容易错位。
- 每个文件经 summarise(across(where(is.numeric), mean)) 会保留多个数值列（如果原文件有多列数值），但你最后用 colnames(expr) <- names(data_list) 只按样本数重命名，列数和样本数不一致就会错配。
- 多文件中同名列在 merge 后可能产生 .x/.y 后缀或被覆盖，进一步混乱。
- 合并为全外连接后会有 NA，但没有处理和检查；列顺序也未明确对齐到样本名顺序。

核心目标是：每个文件最终变成“gene_id + 该样本唯一数值列”，然后按 gene_id 全外连接；最后把 gene_id 设为行名，列名严格对应样本名。

## 稳健的合并方案（可直接运行）

假设每个 CSV 至少有一列数值；若有多列数值，我们对这些数值取行均值作为该样本单一指标（你也可以改成选择指定列，比如 counts/TPM）。

```r
# 路径与文件
path <- "/Users/yuzuan/Downloads/GSE81579_RAW"
files <- list.files(path, pattern = "\\.csv\\.gz$", full.names = TRUE)

# 样本名（去掉 .csv.gz 后缀）
sample_names <- sub("\\.csv\\.gz$", "", basename(files))

# 依赖
library(readr)
library(dplyr)
library(purrr)
library(tidyr)  # 为 column_to_rownames

# 读取并处理单个文件：聚合到 gene_id 唯一，数值列取均值 -> 折叠成单列，列名=样本名
read_one <- function(f, sample) {
  df <- readr::read_csv(f, show_col_types = FALSE)

  if (!"gene_id" %in% names(df)) {
    stop("文件缺少 gene_id 列: ", basename(f))
  }

  df <- df %>%
    mutate(gene_id = as.character(gene_id)) %>%
    group_by(gene_id) %>%
    summarise(across(where(is.numeric), ~ mean(.x, na.rm = TRUE)), .groups = "drop")

  num_df <- df %>% select(where(is.numeric))
  if (ncol(num_df) == 0) {
    stop("文件没有数值列: ", basename(f))
  }

  # 若有多列数值，取行均值；只有一列则直接用
  value <- if (ncol(num_df) == 1) num_df[[1]] else rowMeans(as.matrix(num_df), na.rm = TRUE)

  tibble(gene_id = df$gene_id, !!sample := value)
}

# 批量读取并标准化为 gene_id + 单样本列
data_list_clean <- map2(files, sample_names, read_one)

# 按 gene_id 全外连接成宽表；并确保列顺序与 sample_names 对齐
expr_tbl <- reduce(data_list_clean, full_join, by = "gene_id") %>%
  select(gene_id, all_of(sample_names)) %>%  # 列顺序固定
  distinct(gene_id, .keep_all = TRUE)

# 转为矩阵：行名=gene_id，列名=样本
expr_mat <- expr_tbl %>%
  column_to_rownames("gene_id") %>%
  as.matrix()

# 若需要把 NA 填零（可选）
# expr_mat[is.na(expr_mat)] <- 0
```

备注与假设：
- 假设每个文件有 `gene_id` 列，以及至少一列数值。如果你希望用特定数值列（比如 “counts” 或 “TPM”），把 `rowMeans` 那段改成选定列即可。
- 使用 `full_join` 是“保留所有基因”的合并策略；如果你更想要“只保留各文件共同基因”，把 `reduce(full_join, ...)` 改为 `reduce(inner_join, ...)`。

```r
library(data.table)
files <- list.files(path, pattern = "\\.csv\\.gz$", full.names = TRUE)
sample_names <- sub("\\.csv\\.gz$", "", basename(files))

read_one_dt <- function(f, sample) {
  dt <- fread(f)
  stopifnot("gene_id" %in% names(dt))
  # 聚合到 gene_id 唯一，数值列取均值
  num_cols <- names(dt)[sapply(dt, is.numeric)]
  dt <- dt[, lapply(.SD, function(x) mean(x, na.rm = TRUE)), by = gene_id, .SDcols = num_cols]
  # 折叠为单列（若多数值列）
  vals <- if (length(num_cols) == 1) dt[[num_cols]] else rowMeans(as.matrix(dt[, ..num_cols]), na.rm = TRUE)
  data.table(gene_id = dt$gene_id, tmp = vals)[, setnames(.SD, "tmp", sample)]
}

lst <- lapply(seq_along(files), function(i) read_one_dt(files[i], sample_names[i]))
expr_dt <- Reduce(function(x, y) merge(x, y, by = "gene_id", all = TRUE), lst)
expr_mat <- as.matrix(as.data.frame(expr_dt, row.names = expr_dt$gene_id)[, sample_names, drop = FALSE])
```

- 健壮性小建议：
  - 在读入时加断言：必须有 `gene_id` 与至少一列数值。
  - 明确 inner/full 合并策略，避免意外丢基因或引入大量 NA。
  - 大批量缺失时，确认是否因为不同基因命名空间（如 Ensembl vs Symbol）导致，必要时先做 ID 映射。
