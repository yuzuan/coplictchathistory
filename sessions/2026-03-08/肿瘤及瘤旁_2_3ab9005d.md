这是 **`pandas.arrays.ArrowStringArray` 与 anndata h5ad writer 不兼容**的问题。

**根因**：`adata.obs.index` 的底层类型是 `ArrowStringArray`（pandas 2.x 引入的 `string[pyarrow]` dtype），而当前版本的 anndata 的 HDF5 写入器没有注册对这种类型的序列化方法。

**报错路径**：`adata.write()` → 写 `obs` DataFrame → 写 `_index` 列 → 发现 index 是 `ArrowStringArray` → 找不到对应的写入方法 → 抛出 `IORegistryError`。

**修复方法**：在 `adata.write()` 之前将 index 和 string 列转为原生字符串：

```
adata.obs.index = adata.obs.index.astype(str)
for col in adata.obs.select_dtypes(include=['string']).columns:
    adata.obs[col] = adata.obs[col].astype(str)
adata.write('./write/adata.h5ad')
```

| 配置 | 值 | 影响 |
|------|-----|------|
| `pandas` | **3.0.0** | 默认启用 `future.infer_string=True` |
| `future.infer_string` | **True** | 所有字符串列自动使用 `ArrowStringArray` |
| `pyarrow` | 20.0.0 | 提供 Arrow 后端 |
| `anndata` | 0.12.6 | h5ad writer 尚未支持 `ArrowStringArray` |

**发生了什么**：
- pandas 2.x 时代这个选项默认为 `False`，字符串用 `object` dtype（NumPy）
- **pandas 3.0** 将 `future.infer_string` 默认改为 `True`，所有字符串列自动变为 `string[pyarrow_numpy]`（底层 `ArrowStringArray`）
- `pd.concat`、读 CSV、甚至简单的字符串赋值都会产生 Arrow 字符串
- anndata 0.12.6 的 h5ad writer 还没跟上这个变化

**两种解决方案**：

1. **治标**（每次 write 前转换）：

```
   adata.obs.index = adata.obs.index.astype(str)
   for col in adata.obs.select_dtypes(include=['string']).columns:
       adata.obs[col] = adata.obs[col].astype(str)
```

2. **治本**（在 notebook 开头全局禁用）：

```
   import pandas as pd
   pd.options.future.infer_string = False
```

这样整个 session 中字符串将回退到 `object` dtype，不会再出现 `ArrowStringArray`。

确认了。问题的直接原因是 **pandas 3.0.0** 的行为变更：

| 配置 | 值 | 影响 |
|------|-----|------|
| `pandas` | **3.0.0** | 默认启用 `future.infer_string=True` |
| `future.infer_string` | **True** | 所有字符串列自动使用 `ArrowStringArray` |
| `pyarrow` | 20.0.0 | 提供 Arrow 后端 |
| `anndata` | 0.12.6 | h5ad writer 尚未支持 `ArrowStringArray` |

**发生了什么**：
- pandas 2.x 时代这个选项默认为 `False`，字符串用 `object` dtype（NumPy）
- **pandas 3.0** 将 `future.infer_string` 默认改为 `True`，所有字符串列自动变为 `string[pyarrow_numpy]`（底层 `ArrowStringArray`）
- `pd.concat`、读 CSV、甚至简单的字符串赋值都会产生 Arrow 字符串
- anndata 0.12.6 的 h5ad writer 还没跟上这个变化

**两种解决方案**：

1. **治标**（每次 write 前转换）：

```
   adata.obs.index = adata.obs.index.astype(str)
   for col in adata.obs.select_dtypes(include=['string']).columns:
       adata.obs[col] = adata.obs[col].astype(str)
```

2. **治本**（在 notebook 开头全局禁用）：

```
   import pandas as pd
   pd.options.future.infer_string = False
```

这样整个 session 中字符串将回退到 `object` dtype，不会再出现 `ArrowStringArray`。
