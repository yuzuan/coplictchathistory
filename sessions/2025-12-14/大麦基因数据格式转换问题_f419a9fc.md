### 🤖 Copilot <sub>2025/12/14 20:05</sub>

这类 `HORVU.MOREX.r2.1HG0000010` 基本上是（Morex v2）注释体系里的基因ID，**不能靠规则直接“算出”NCBI Gene**，必须走“对照表/数据库映射”。最稳的一条路是用 NCBI 的 `gene2ensembl` + `gene_info` 生成映射（大麦 taxid 通常是 **4513**）。

下面给你一套可直接在 macOS 终端跑的流程：把 `HORVU...` 映射成 **NCBI 的 Gene Symbol（通常是 `LOC#########`）**，同时也保留 **GeneID 数字**。

## 1) 下载 NCBI 对照数据
```bash
curl -L -o gene2ensembl.gz https://ftp.ncbi.nlm.nih.gov/gene/DATA/gene2ensembl.gz
curl -L -o gene_info.gz    https://ftp.ncbi.nlm.nih.gov/gene/DATA/gene_info.gz
```

## 2) 生成 “HORVU… → GeneID → Symbol(LOC…)” 映射表
```bash
# 4513 = Hordeum vulgare（大麦）；如果你用的不是这个物种/亚种，先改 taxid
TAXID=4513

# gene2ensembl: tax_id, GeneID, Ensembl_gene_identifier, Ensembl_rna_identifier, Ensembl_protein_identifier
zgrep -P "^${TAXID}\t" gene2ensembl.gz | cut -f2,3 \
  | awk -F'\t' '{print $2"\t"$1}' > ensemblGene_to_geneid.tsv
# 输出：EnsemblGeneID(HORVU...) \t GeneID

# gene_info: tax_id, GeneID, Symbol, ...
zgrep -P "^${TAXID}\t" gene_info.gz | cut -f2,3 > geneid_to_symbol.tsv
# 输出：GeneID \t Symbol(很多就是LOCxxxx)

LC_ALL=C sort -k2,2 ensemblGene_to_geneid.tsv > ensemblGene_to_geneid.sorted.tsv
LC_ALL=C sort -k1,1 geneid_to_symbol.tsv      > geneid_to_symbol.sorted.tsv

join -t $'\t' -1 2 -2 1 ensemblGene_to_geneid.sorted.tsv geneid_to_symbol.sorted.tsv \
  | awk -F'\t' '{print $2"\t"$3"\t"$1}' > horvu_to_ncbi.tsv
# 输出：HORVU... \t Symbol(LOC...) \t GeneID
```

## 3) 批量转换你的基因列表
假设你有个 `ids.txt`（第一列就是 HORVU 基因ID，一行一个）：
```bash
awk -F'\t' 'NR==FNR{m[$1]=$2"\t"$3; next}
            {print $1"\t"(m[$1]?m[$1]:"NA\tNA")}' horvu_to_ncbi.tsv ids.txt \
  > ids_mapped.tsv
# 输出：HORVU... \t LOC... \t GeneID（没命中的给 NA）
```

## 重要说明（避免踩坑）
- `gene2ensembl` 里第三列叫 *Ensembl_gene_identifier*，但很多物种会放入“Ensembl系/官方注释系”的基因ID；你的 `HORVU.MOREX...` 是否在里面，跑完第 2 步就知道了。
- NCBI “GeneID” 是数字；`LOC#########` 通常是该 GeneID 对应的 **Symbol**，但也可能出现不是 LOC 开头的正规符号，所以我这里用 `gene_info` 的 Symbol 作为最终“NCBI Gene 名称”。
