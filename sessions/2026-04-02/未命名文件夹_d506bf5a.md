### 变更含义

| 版本 | 行为 |
|---|---|
| **v1.8.2** | `getGEO2()` 返回 list 时，会自动提取 `$sample_anno` 元素，只返回样本注释表 |
| **v1.9.0** | 不再自动提取，直接返回 `getGEO2()` 的原始返回值（可能是 list/data.frame 等） |

这意味着：
- 如果 `getGEO2()` 返回的是一个包含多个元素的 list（比如多平台数据集），**v1.9.0 会保留完整的 list 结构**返回给用户，而不是只取 `sample_anno`
- 这是一个**破坏性变更（breaking change）**：依赖旧版 `get_GEO_pheno()` 直接拿到 data.frame 的代码，升级后可能拿到 list，需要用户自己判断和提取

###

— `<<-` 修复

**v1.8.2**（第 161 行）：

```
anno_obj <<- c(anno_obj, res)
```

**v1.9.0**（第 161 行）：

```
anno_obj <- c(anno_obj, res)
```

变更意义：`<<-` 是 R 的全局赋值操作符，会修改父环境中的 `anno_obj`。改为 `<-` 后只在当前环境赋值，**但这也意味着下面 `save(anno_obj, ...)` 保存的仍然是更新后的本地副本**，功能等价但更安全——不再意外污染全局环境。
