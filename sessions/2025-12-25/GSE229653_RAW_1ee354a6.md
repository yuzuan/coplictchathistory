## 主要"表达量"指标

### 1. **ReadCount** ⭐ (最核心)
- **含义**：在该基因组位置检测到的测序 reads 数量
- **代表**：蛋白-RNA 结合的**强度/丰度**
- **类似于**：RNA-seq 中的 read counts
- **用途**：
  - ReadCount 越高 = 结合位点信号越强
  - 用于样本间比较（需标准化）
  - 差异结合分析的基础

### 2. **ConversionEventCount** (重要辅助指标)
- **含义**：交联诱导的碱基转换事件数
- **代表**：蛋白-RNA **直接交联的证据**
- **特点**：更精确定位真实结合位点
- **质量控制**：高转换事件 = 高置信度的结合

### 3. **ModeScore** (置信度指标)
- **含义**：结合位点的置信度/质量分数（0-1之间）
- **用途**：过滤低质量的 clusters

## 数据分析建议

```r
# 核心指标使用示例：

# 1. 基础过滤（推荐）
filtered_data <- data[ReadCount >= 5 & ModeScore >= 0.7]

# 2. 样本间比较（使用 ReadCount）
# 需要标准化，常用方法：
# - CPM (Counts Per Million)
# - TPM 类似方法
# - DESeq2 标准化

# 3. 识别强结合位点
strong_binding <- data[ReadCount >= 20 & ConversionEventCount >= 10]

# 4. 计算富集倍数（相对于 IgG 对照）
# Fold_change = ReadCount_IGF2BP3 / ReadCount_IgG
```

## 类比理解

| 数据类型 | "表达量"指标 | 含义 |
|---------|------------|------|
| RNA-seq | Read counts | 基因表达水平 |
| ChIP-seq | Peak intensity | 蛋白-DNA结合强度 |
| **eCLIP-seq** | **ReadCount** | **蛋白-RNA结合强度** |

**总结**：在宽格式合并时，最重要的是保留每个样本的 **ReadCount** 列，这是核心的定量数据。ConversionEventCount 和 ModeScore 可作为质量控制的辅助指标。
