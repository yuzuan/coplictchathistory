## 阿里云开放端口教程

### 方法一：通过阿里云控制台配置安全组

1. **登录阿里云控制台**
   - 访问 https://ecs.console.aliyun.com

2. **找到安全组**
   - 左侧菜单：**网络与安全** → **安全组**
   - 或者进入实例详情页 → 点击 **安全组** 标签

3. **配置入方向规则**
   - 点击安全组 → **管理规则** → **入方向** → **手动添加**

添加以下规则：

| 授权策略 | 优先级 | 协议类型 | 端口范围 | 授权对象 |
   |---------|-------|---------|---------|---------|
   | 允许 | 1 | TCP | 80/80 | 0.0.0.0/0 |
   | 允许 | 1 | TCP | 443/443 | 0.0.0.0/0 |
   | 允许 | 1 | TCP | 3000/3000 | 0.0.0.0/0 |

4. **保存规则**

---

### 方法二：服务器内部防火墙设置

如果服务器内部有防火墙，也需要开放端口：

**Ubuntu/Debian (ufw):**

```
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 3000/tcp
sudo ufw reload
```

**CentOS/RHEL (firewalld):**

```
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --permanent --add-port=443/tcp
sudo firewall-cmd --permanent --add-port=3000/tcp
sudo firewall-cmd --reload
```

**CentOS 6 (iptables):**

```
sudo iptables -I INPUT -p tcp --dport 80 -j ACCEPT
sudo iptables -I INPUT -p tcp --dport 443 -j ACCEPT
sudo iptables -I INPUT -p tcp --dport 3000 -j ACCEPT
sudo service iptables save
```

---

### 验证端口是否开放

在本地电脑执行：

```
# 检测端口是否开放
telnet 你的服务器IP 3000

# 或者用 nc
nc -zv 你的服务器IP 3000
```

## 飞牛OS (fnOS) 部署指南

飞牛OS 是一款国产 NAS 操作系统，主要有两种部署方式：

---

### 方式一：物理机安装（推荐）

飞牛OS 设计为直接安装在物理机上作为 NAS 系统使用。

**1. 下载镜像**
- 官网：https://www.fnnas.com
- 下载最新的 ISO 镜像文件

**2. 制作启动盘**

```
# macOS/Linux 使用 dd 命令
sudo dd if=fnos-xxx.iso of=/dev/sdX bs=4M status=progress

# 或使用 balenaEtcher、Rufus 等工具
```

**3. 安装步骤**
- U盘启动 → 选择安装磁盘 → 等待安装完成
- 安装完成后通过浏览器访问 `http://NAS的IP:5666`

---

### 方式二：虚拟机安装

**VMware/Proxmox/ESXi 安装：**

1. 创建虚拟机：
   - 类型：Linux 64位
   - 内存：至少 4GB（推荐 8GB+）
   - 硬盘：系统盘 32GB + 数据盘若干
   - 网络：桥接模式

2. 挂载 ISO 镜像启动安装

---

### 方式三：Docker 部署（仅部分服务）

飞牛OS 本身不支持 Docker 部署，但你可以在云服务器上用 Docker 部署类似的 NAS 服务：

**替代方案 - CasaOS（轻量 NAS 系统）：**

```
curl -fsSL https://get.casaos.io | sudo bash
```

**替代方案 - Nextcloud（私有云盘）：**

```
docker run -d \
  --name nextcloud \
  -p 8080:80 \
  -v nextcloud_data:/var/www/html \
  nextcloud
```

---

### 阿里云服务器注意事项

⚠️ **云服务器不太适合部署飞牛OS**，原因：

1. **飞牛OS 是完整操作系统**，需要裸机安装，不是应用程序
2. **云服务器无法直接安装第三方OS**（除非使用自定义镜像）
3. **存储成本高**：NAS 需要大容量存储，云盘价格昂贵

---

### 推荐方案

| 需求 | 推荐方案 |
|-----|---------|
| 家用 NAS | 物理机安装飞牛OS / 群晖 |
| 云端文件同步 | Nextcloud / Seafile |
| 轻量 NAS 体验 | CasaOS / OpenMediaVault |
| 影音媒体库 | Jellyfin / Plex |

## 配置 HTTP/HTTPS 代理方法

根据你的部署方式，有几种配置代理的方法：

---

### 方法一：Docker 环境变量配置（推荐）

如果你是用 Docker 部署的，在 `docker-compose.yml` 中添加：

```
version: '3'
services:
  your-app:
    image: your-image
    environment:
      - HTTP_PROXY=http://代理IP:端口
      - HTTPS_PROXY=http://代理IP:端口
      - http_proxy=http://代理IP:端口
      - https_proxy=http://代理IP:端口
      # 如果代理需要认证
      # - HTTP_PROXY=http://用户名:密码@代理IP:端口
```

或者用 `docker run` 命令：

```
docker run -d \
  -e HTTP_PROXY=http://代理IP:端口 \
  -e HTTPS_PROXY=http://代理IP:端口 \
  -e http_proxy=http://代理IP:端口 \
  -e https_proxy=http://代理IP:端口 \
  your-image
```

---

### 方法二：修改应用配置文件

如果应用有 `.env` 配置文件，直接编辑：

```
# 进入容器或服务器
vi /app/config/env  # 根据实际路径

# 添加以下内容
HTTP_PROXY=http://代理IP:端口
HTTPS_PROXY=http://代理IP:端口
```

---

### 方法三：系统环境变量（Linux）

```
# 临时生效（当前会话）
export HTTP_PROXY=http://代理IP:端口
export HTTPS_PROXY=http://代理IP:端口
export http_proxy=http://代理IP:端口
export https_proxy=http://代理IP:端口

# 永久生效 - 添加到 ~/.bashrc 或 /etc/environment
echo 'export HTTP_PROXY=http://代理IP:端口' >> ~/.bashrc
echo 'export HTTPS_PROXY=http://代理IP:端口' >> ~/.bashrc
source ~/.bashrc
```

---

### 方法一：Docker 单容器部署（推荐）

通过 SSH 或宝塔终端执行：

```
# 1. 创建数据目录
mkdir -p /opt/cloudsaver/data
mkdir -p /opt/cloudsaver/config

# 2. 运行容器
docker run -d \
  -p 8008:8008 \
  -v /opt/cloudsaver/data:/app/data \
  -v /opt/cloudsaver/config:/app/config \
  --name cloud-saver \
  --restart unless-stopped \
  jiangrui1994/cloudsaver:latest
```

---

### 方法二：Docker Compose 部署

```
# 1. 创建目录
mkdir -p /opt/cloudsaver && cd /opt/cloudsaver

# 2. 创建 docker-compose.yml
cat > docker-compose.yml << 'EOF'
version: "3"
services:
  cloudsaver:
    image: jiangrui1994/cloudsaver:latest
    container_name: cloud-saver
    ports:
      - "8008:8008"
    volumes:
      - ./data:/app/data
      - ./config:/app/config
    restart: unless-stopped
EOF

# 3. 启动
docker-compose up -d
```

## 代理 IP 和端口的获取方式

你需要先有一个代理服务器，才能填写代理地址。以下是几种获取方式：

---

### 方式一：使用机场服务（最简单）

如果你有订阅机场（VPN/代理服务），通常可以获取 HTTP 代理地址：

1. 打开你的代理客户端（如 Clash、V2rayN、Shadowrocket 等）
2. 查看设置中的 **HTTP 代理端口**
3. 格式通常是：`http://127.0.0.1:7890`

**但是！** 这个本地代理在云服务器上用不了，你需要：
- 在云服务器上也安装代理客户端，或
- 使用机场提供的**直连代理地址**（部分机场提供）

---

### 方式二：自建代理服务器（推荐）

在香港/海外服务器上搭建代理，然后云服务器连接它：

**1. 购买海外 VPS**（香港、新加坡等）
- 推荐：腾讯云香港、阿里云香港、Vultr、搬瓦工等
- 约 ￥30-50/月

**2. 在海外 VPS 上安装代理**

安装 [3x-ui](https://github.com/MHSanaei/3x-ui)（一键脚本）：

```
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

或安装 Squid HTTP 代理：

```
# Ubuntu/Debian
apt update && apt install squid -y

# 配置允许你的阿里云IP访问
echo "acl aliyun src 8.152.203.38" >> /etc/squid/squid.conf
echo "http_access allow aliyun" >> /etc/squid/squid.conf
systemctl restart squid
```

**3. 填写代理地址**

```
HTTP_PROXY=http://海外VPS的IP:3128
```

### 第二步：查看 HTTP 代理端口

#### Clash 系列（Clash Verge / ClashX / CFW）

1. 打开 Clash 客户端
2. 查看 **端口设置**：
   - 混合端口（Mixed Port）：通常是 `7890`
   - HTTP 端口：通常是 `7890`
3. 代理地址就是：`http://127.0.0.1:7890`

#### V2rayN

1. 打开 V2rayN
2. 主界面底部会显示：`本地HTTP代理: 127.0.0.1:10809`
3. 代理地址：`http://127.0.0.1:10809`

**本地代理（127.0.0.1）无法直接给云服务器用！**

mv clash-linux-amd64-v1.18.0 clash
chmod +x clash

mkdir -p /root/.config/clash

# 4. 后台运行 Clash

## VS Code 连接云服务器教程

### 方法一：使用 Remote-SSH 扩展（推荐）

#### 1. 安装扩展

在 VS Code 中安装 **Remote - SSH** 扩展：
- 按 `Cmd+Shift+X` 打开扩展商店
- 搜索 `Remote - SSH`
- 安装 Microsoft 官方的扩展

#### 2. 配置 SSH 连接

按 `Cmd+Shift+P`，输入 `Remote-SSH: Connect to Host...`，选择 **Add New SSH Host**

输入：

```
ssh root@8.152.203.38
```

选择保存到 `~/.ssh/config`

#### 3. 连接服务器

- 按 `Cmd+Shift+P` → `Remote-SSH: Connect to Host...`
- 选择 `8.152.203.38`
- 输入服务器密码
- 等待连接完成

#### 4. 打开文件夹

连接成功后，点击 **打开文件夹**，输入路径如：

```
/opt/cloudsaver
```

---

### 方法二：配置免密登录（可选，更方便）

在你的 Mac 终端执行：

```
# 生成 SSH 密钥（如果没有的话）
ssh-keygen -t rsa -b 4096

# 复制公钥到服务器
ssh-copy-id root@8.152.203.38
```

之后连接就不需要输入密码了。

---

### SSH 配置文件参考

编辑 `~/.ssh/config`，添加：

```
Host aliyun
    HostName 8.152.203.38
    User root
    Port 22
```

之后在 VS Code 中可以直接选择 `aliyun` 连接。

---

连接成功后，你可以：
- 直接在 VS Code 中编辑服务器文件
- 打开终端执行命令（`` Ctrl+` ``）
- 继续配置 ShellClash

## 查看/重置阿里云服务器密码

### 方法一：在阿里云控制台重置密码

如果你忘记了密码，可以重置：

1. 登录 [阿里云控制台](https://ecs.console.aliyun.com)
2. 找到你的 ECS 实例
3. 点击 **更多** → **密码/密钥** → **重置实例密码**
4. 设置新密码
5. **重启实例**使密码生效

---

### 方法二：在宝塔面板查看

如果你是通过宝塔面板登录的，宝塔面板本身有 SSH 终端功能：

1. 登录宝塔面板
2. 左侧菜单 → **终端**
3. 直接使用，不需要密码

---

### 方法三：检查购买时的信息

- 查看购买服务器时阿里云发送的**短信/邮件**
- 里面可能包含初始密码

---

### 方法四：使用 SSH 密钥登录（推荐）

如果你之前设置过密钥：

1. 阿里云控制台 → ECS → **网络与安全** → **密钥对**
2. 下载私钥文件（`.pem`）
3. 在 Mac 上配置：

```
# 将私钥放到 ~/.ssh/ 目录
mv ~/Downloads/your-key.pem ~/.ssh/
chmod 400 ~/.ssh/your-key.pem

# 配置 SSH
echo "
Host aliyun
    HostName 8.152.203.38
    User root
    IdentityFile ~/.ssh/your-key.pem
" >> ~/.ssh/config
```

---

### 最简单的方法

既然你已经能通过**宝塔面板的终端**操作服务器，说明宝塔已经有权限了。

你可以：
1. 在宝塔终端中重置 root 密码：

```
   passwd root
```

2. 输入两次新密码
3. 然后用新密码连接 VS Code
