postscriptFonts(sans = postscriptFonts()[["Helvetica"]])

唯一改动：第 7-8 行加了 `pdfFonts()` / `postscriptFonts()` 映射，其余完全不变。 `pdfFonts()` 映射无效，说明 macOS 中文 locale 下 R 的 PDF 设备字体解析绕过了这层映射。有两个可靠方案：

---

**方案 A（推荐）：`showtext` 接管字体渲染**

在所有代码最前面加：

```r
library(showtext)
showtext_auto()
```

如果未安装：`install.packages("showtext")`

---

**方案 B（不装新包）：全部替换为 `"Helvetica"`**

`"sans"` 在 macOS 中文 locale 会被系统解析为 `"Heiti SC"`，而 `"Helvetica"` 是 PDF 设备内建字体，不经过系统映射。把代码中所有 `fontfamily = "sans"` 和 `base_family = "sans"` 和 `family = "sans"` 全替换为 `"Helvetica"`：

```r
par(family = "Helvetica")

# ggplot
theme_bw(base_family = "Helvetica")
family = "Helvetica"            # geom_text_repel

# ComplexHeatmap gpar
gpar(fontfamily = "Helvetica")
```

另外 `HeatmapAnnotation` 里的 `gp = gpar(fontfamily = "sans")` 控制的是注释色块而非文字，可以直接删掉：

```r
ha_deg <- HeatmapAnnotation(

    show_annotation_name = FALSE,
    annotation_legend_param = list(Group = list(title = NULL))
```
