## mambaforge vs miniconda3

| | **miniconda3** | **mambaforge** |
|---|---|---|
| **维护者** | Anaconda Inc. | conda-forge 社区 |
| **默认 channel** | defaults（Anaconda 商业源） | conda-forge（社区开源源） |
| **包管理器** | conda（纯 Python，较慢） | mamba（C++ 重写的 conda，快 5-10x） |
| **许可** | defaults channel 商业使用需付费（>200人组织） | conda-forge 完全免费 |
| **solver** | conda 经典 solver（慢） | libmamba solver（快） |

### 你的系统情况

- **mambaforge**（

）— 通过 Homebrew Cask 安装，管理了你大部分环境（sc, sc2, ena, p2p 等）
- **miniconda3**（

### 实际影响

两者创建的环境**完全兼容**，都是标准 conda 环境。主要区别：
- 用 mamba install 比 conda install 解依赖快很多
- mambaforge 默认从 conda-forge 拉包，版本通常更新更快
- 你当前 shell 激活的是 miniconda3（* 标记），但通过 conda activate sc 可以切换到 mambaforge 管理的环境
