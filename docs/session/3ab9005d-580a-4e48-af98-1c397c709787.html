| 配置 | 值 | 影响 |
|------|-----|------|
| pandas | **3.0.0** | 默认启用 future.infer_string=True |
| future.infer_string | **True** | 所有字符串列自动使用 ArrowStringArray |
| pyarrow | 20.0.0 | 提供 Arrow 后端 |
| anndata | 0.12.6 | h5ad writer 尚未支持 ArrowStringArray |

**发生了什么**：
- pandas 2.x 时代这个选项默认为 False，字符串用 object dtype（NumPy）
- **pandas 3.0** 将 future.infer_string 默认改为 True，所有字符串列自动变为 string[pyarrow_numpy]（底层 ArrowStringArray）
- pd.concat、读 CSV、甚至简单的字符串赋值都会产生 Arrow 字符串
- anndata 0.12.6 的 h5ad writer 还没跟上这个变化

**两种解决方案**：

1. **治标**（每次 write 前转换）：

   adata.obs.index = adata.obs.index.astype(str)

2. **治本**（在 notebook 开头全局禁用）：

   import pandas as pd
   pd.options.future.infer_string = False

这样整个 session 中字符串将回退到 object dtype，不会再出现 ArrowStringArray。
