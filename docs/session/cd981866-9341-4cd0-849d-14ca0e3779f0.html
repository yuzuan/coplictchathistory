如何显示每个 cell 的位置

Copilot 2026/02/25 16:54

在 VS Code 的 Jupyter Notebook 中，有几种方式查看每个 Cell 的位置：

---

**方法 1：查看执行序号（最常用）**

每个 Cell 左侧的 [ ] 括号内会显示**执行顺序编号**（运行后显示数字，如 [3]），但这是执行顺序，不是位置序号。

---

**方法 2：开启行号**

快捷键 Shift + L（光标在 Cell 内时）— 为当前 Cell 切换行号显示。

全局开启：Cmd + Shift + P → 搜索 Notebook: Toggle Cell Line Numbers

---

**方法 3：大纲视图（最直观）**

Cmd + Shift + P → View: Open Outline → 侧边栏显示所有 Cell 及其序号，可点击跳转。

---

**方法 4：底部状态栏**

点击任意 Cell 后，VS Code 底部状态栏右侧会显示类似 Cell 14 of 58 的位置信息，这与我说的 Cell 编号一一对应。

---

所以我说的"Cell 14"就对应状态栏显示的 Cell 14 of 58，这是最快的定位方式。

为什么 0 和 5 在一块呢？

Copilot 2026/02/25 21:24

这是正常的生物学现象，原因有两点：

**1. UMAP 距离 = 转录组相似度**

UMAP 是按全局基因表达相似性降维的，簇 0（Phago/AP-TAM）和簇 5（M2-like-TAM）虽然 top markers 不同，但它们共享大量背景基因（都是单核细胞来源的 TAM，都表达 CD163、LYZ、AIF1 等髓系基因），整体转录组轮廓接近，所以落在相邻区域。

**2. Leiden 是在高维空间切割，而非 UMAP 上切割**

Leiden 聚类在 PCA/Harmony 空间的 kNN 图上运行，它发现的是高维邻居关系中的局部社区结构。两个簇可以在 30 维 PCA 空间里被清晰分开，但投影到 2D UMAP 后显得重叠——这是**降维失真**，不是聚类错误。

**生物学解释：**

| 簇 | 主要特征 | 共同点 |
|---|---|---|
| 0 Phago/AP-TAM | CD83↑, CCL3↑, 早期激活 | 都是 Mo-TAM |
| 5 M2-like-TAM | CD163↑, MRC1↑, 抗炎极化 | 背景髓系基因相似 |

这两个状态在肿瘤微环境中可以相互转化（激活→抗炎极化），转录组上存在连续谱，Leiden 以 resolution=0.5 把它们切成两个簇是合理的，但它们不像 Cluster 8（Microglia）那样与主体完全分离。

如果觉得这两个簇没必要区分，可以把 resolution 降低，它们会合并为一个簇。
