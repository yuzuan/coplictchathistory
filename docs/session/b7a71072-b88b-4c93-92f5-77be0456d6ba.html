简短结论：错误来自 scanpy 在用 numba 计算稀疏矩阵的均值/方差时，遇到了非数值/对象类型的数据（例如 dtype=object），numba 无法把 Python 对象转换成本地数值类型，从而抛出 “can't unbox array from PyObject into native value” 的 TypeError。

常见原因与定位方法：
- adata.X（或稀疏矩阵的 adata.X.data）含有 dtype=object 或包含非数值值。
- 或者在构建 AnnData 时没有把读入的矩阵转换为数值 dtype（例如 float32/float64）。
- 触发点在 sc.pp.calculate_qc_metrics() 内部的 numba 实现，对输入数组类型要求严格。

快速检查和修复（在 notebook 中运行）：

        else:
            ad.X = ad.X.astype(np.float32)

        print(ad.obs["sample"].iloc[0], "array, dtype =", ad.X.dtype)
        if ad.X.dtype == object:
