为什么 Python 代码运行结果和 R 代码运行的结果不一致

Copilot 2025/12/08 16:17

导致结果不一致的主要可能点：

- 统计门槛不同：当前 Python 代码用 FDR < 0.01 且只取 top10/cluster，很多情况下会变成空集直接跳过绘图；R 代码若阈值更宽或取更多条目，会有结果。
- 基因背景不同：Python 固定背景 N=20000 计算超几何；R 脚本可能用实际检测到的基因或其他背景，富集 p 值会不同。
- 输入基因集差异：仅使用上调且 p<0.01 的基因；若 R 侧用双向或不同阈值，富集列表就会不同。
- 排序/去重策略差异：Python 里按每列 nsmallest(10) 的顺序合并；R 可能按整体排序或去重方式不同。
- 聚类/缩放默认值差异：seaborn::clustermap 与 R 的 pheatmap/ComplexHeatmap 在距离度量、缺失值处理、缩放方式上默认不同，会导致行列顺序和显示不同。

对齐建议（可任选几条尝试）：
1) 放宽阈值以接近 R：例如 FDR < 0.05；或在 p 值过滤前保留更多基因（如 p<0.05，logFC>0）。
2) 用实际背景：将 N_tot 换成 go_raw 里的唯一基因数，或 clusters 中所有基因的并集大小。
3) 统一选择规则：改成全局排序取前 N（如全表最小的 50 个 term），或每列 top20，再合并去重。
4) 统一距离/聚类：在 clustermap 里显式设 metric="euclidean", method="average"（或与 R 相同的设置），并在 log_df 前对行/列 z-score 处理与 R 保持一致。
5) 生成中间文件对比：导出 Python 的 p_df/log_df，与 R 侧对应数据框对比差异，先看哪些 term 和 p 值不同，再定位是输入基因还是背景/阈值造成。
