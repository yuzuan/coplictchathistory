Copilot 2026/02/02 00:03

这两者的主要区别在于：

## 1. **算法不同**
- **Python (Scanpy)**: 使用 **Leiden 算法**

  sc.tl.leiden(adata_combined, resolution=0.5)

- **R (Seurat)**: 默认使用 **Louvain 算法**（也支持 Leiden）

  FindClusters(object, resolution = c(0.1, 0.5, 0.8))

## 2. **分辨率含义略有差异**
- 虽然都是控制聚类粒度，但 **相同数值的分辨率在两种算法中不完全等价**
- Leiden 算法通常能产生更优化的聚类结果

## 3. **多分辨率测试**
R 中的做法是**同时测试多个分辨率**并保存：

# Seurat 会自动创建多个列
RNA_snn_res.0.1  # 分辨率 0.1
RNA_snn_res.0.5  # 分辨率 0.5
RNA_snn_res.0.8  # 分辨率 0.8

在 Python 中可以这样实现：

# 测试多个分辨率
for res in [0.1, 0.5, 0.8, 1.0]:
    sc.tl.leiden(
        adata_combined, 
        resolution=res,
        key_added=f"leiden_res_{res}"
    )
    print(f"分辨率 {res}: {adata_combined.obs[f'leiden_res_{res}'].nunique()} 个聚类")

## 4. **推荐做法**

如果您想在 Python 中对比不同分辨率的效果（类似 R 的做法），可以添加这段代码：

# 测试多个分辨率并可视化对比
resolutions = [0.1, 0.5, 0.8, 1.0]
for res in resolutions:
    sc.tl.leiden(adata_combined, resolution=res, key_added=f"leiden_res_{res}")

# 可视化对比
sc.pl.umap(adata_combined, color=[f"leiden_res_{res}" for res in resolutions])

您当前的 resolution=0.5 通常是一个**中等粒度**的选择，适合初步探索数据。
