## snRNA-seq vs scRNA-seq 核心区别

| 维度 | scRNA-seq（单细胞） | snRNA-seq（单核） |
|------|---------------------|-------------------|
| **输入物质** | 完整活细胞 | 从细胞中提取的细胞核 |
| **组织解离** | 需要酶消化成单细胞悬液 | 温和裂解获取细胞核，无需完整解离 |
| **检测的 RNA** | 细胞质 mRNA 为主（成熟转录本） | 核内 RNA（含 pre-mRNA、内含子读段比例高） |
| **适用组织** | 容易解离的组织（血液、肿瘤） | **冻存/难解离组织**（脑、脂肪、肌肉） |
| **细胞类型偏差** | 大细胞（神经元）容易丢失；解离敏感基因上调 | 无解离偏差，**神经元等脆弱细胞保留更好** |
| **线粒体基因** | 高（细胞质丰富） | 极低（线粒体在核外） |
| **基因检出量** | 每细胞略高（~2000-5000 genes） | 每核略低（~1000-3000 genes） |

### 对你项目的影响

1. **你现有的 GSE162631 和 GSE273274 都是 scRNA-seq**（完整细胞），而 **GSE274987 是 snRNA-seq**（单核）
2. **整合是可行的**，但需要注意：
- snRNA-seq 内含子比例高 → 需确认比对时是否包含 pre-mRNA
- 线粒体 QC 标准不同：scRNA-seq 用 MT% < 20%，snRNA-seq 通常 MT% 本身就极低
- 基因检出量分布不同 → Harmony 批次矫正可以部分缓解
- ambient RNA 特征不同

3. **脑组织特别适合 snRNA-seq**：因为神经元体积大且突起脆弱，scRNA-seq 会严重丢失神经元。GSE274987 用 snRNA-seq 恰好能更好地捕获脑组织中的所有细胞类型（包括神经元和少突胶质细胞）。

### 结论

snRNA-seq 与 scRNA-seq **可以整合**，在 scanpy/Harmony 框架下操作相同，只需在 QC 参数和批次矫正时做适当调整。不过如果你希望数据高度一致，可以优先选择同为 scRNA-seq 的数据集。
