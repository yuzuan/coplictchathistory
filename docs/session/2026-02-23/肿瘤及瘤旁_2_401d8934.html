## 1. 基因过滤时机不当（最重要）

**问题**：fixed_qc 函数在**每个样本**内单独执行 sc.pp.filter_genes(min_cells=3)，然后再用 join="outer" 合并。这导致：
- 各样本保留的基因集不同
- join="outer" 会用 0/NaN 填充缺失基因，后续统计失真

**正确做法**：基因过滤应在合并后的完整数据集上执行一次。

def fixed_qc(adata, qc_cfg=QC_CONFIG):
    # 只做细胞 QC，不在这里过滤基因
    # sc.pp.filter_genes(adata, min_cells=qc_cfg["min_cells"])  ← 移除这行
    ...

# 合并后再过滤基因
sc.pp.filter_genes(adata_combined, min_cells=3)

---

## 2. HVG 未指定 flavor='seurat_v3'

注释中已写明 seurat_v3 flavor，但代码调用时缺少该参数（默认是旧版 'seurat'）：

# 当前代码（有问题）
sc.pp.highly_variable_genes(adata_combined, n_top_genes=HVG_N_TOP, batch_key=BATCH_KEY)

# 修正
sc.pp.highly_variable_genes(
    adata_combined, n_top_genes=HVG_N_TOP,
    flavor='seurat_v3', batch_key=BATCH_KEY, layer='counts'  # seurat_v3 需要原始 counts
)

seurat_v3 要求在原始 counts（未归一化）上计算，应通过 layer='counts' 指定。

---

## 3. 归一化缺少 target_sum=1e4

# 当前代码
sc.pp.normalize_total(adata_combined)

# 修正：指定标准化目标值
sc.pp.normalize_total(adata_combined, target_sum=1e4)

不指定时默认归一化到中位数 library size，与 Seurat CPM 标准不一致。

---

## 4. PCA 前缺少 sc.pp.scale

当前流程直接在 log1p 数据上做 PCA，没有 scale 步骤。高表达基因会主导 PCA：

# 在 PCA 之前添加
sc.pp.scale(adata_combined, max_value=10)

---

## 5. 未保存 adata.raw

代码虽然保存了 layers["counts"]，但没有设置 adata.raw。后续 sc.pl.umap(color=gene, use_raw=True) 会报错：

# 在 highly_variable_genes 之后、subset HVG 之前添加
adata.raw = adata_combined  # 保存完整基因集的 log 数据

---

## 6. sc.concat 的 label="batch" 冗余

代码已有 sample 列用于批次矫正，label="batch" 会新增一个 batch 列（与 sample 完全重复），造成元数据冗余：

adata_combined = sc.concat(
    adatas_to_merge,
    join="inner",   # 建议改为 inner（取共同基因），避免 NaN 问题
    merge="unique",
    # 去掉 label="batch", keys=... 这两行
)

---

| 优先级 | 问题 | 影响 |
|--------|------|------|
| 🔴 高 | 基因过滤在单样本上而非合并后 | 影响基因集一致性，outer join 引入 NaN |
| 🔴 高 | HVG 未用 flavor='seurat_v3' | 选出的 HVG 与预期不同 |

| 🟡 中 | 未设置 adata.raw | 可视化 use_raw=True 时报错 |

当前代码用 log1p 数据算 HVG，两种 flavor 都接受，但结果不同：
