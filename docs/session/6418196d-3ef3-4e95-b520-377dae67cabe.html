问题原因找到了！

**根本问题**：gene_set_name 变量只在两种情况下被定义：
1. 在 MSigDB 的 try 块中成功时定义为 'HALLMARK_ADIPOGENESIS'
2. 在自定义基因集的 if gene_set is None 条件中定义为 'MP_genes'

但如果 MSigDB 下载失败（进入 except 块），gene_set 和 gene_set_name 都不会被定义。然后在 fallback 逻辑中，因为 gene_set 未定义（而不是 None），会直接报 NameError。

需要在开始时初始化这两个变量：

✅ 已修复！

**问题原因**：
- gene_set 初始化为 None，但 gene_set_name 没有初始化
- 当 MSigDB 下载失败时，gene_set_name 从未被定义
- 后续代码使用 f'{gene_set_name} score' 时就会报 NameError

**解决方案**：
- 同时初始化 gene_set = None 和 gene_set_name = None
- 这样即使 MSigDB 失败，fallback 逻辑也能正确将 gene_set_name 设置为 'MP_genes'

现在可以重新运行该单元格了！
