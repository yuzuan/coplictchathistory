macOS 中文 locale 把 "sans" 映射到 "Heiti SC"，而 R 的 PostScript/PDF 设备字体库里没有这个字体。ComplexHeatmap 内部用 pdf(file=NULL) 做尺寸计算时触发了该错误。

在脚本最顶部（library(ComplexHeatmap) 之后、任何绘图之前）加两行即可：

# 强制 PDF/PS 设备将 "sans" 解析为 Helvetica，而非系统 CJK 字体
pdfFonts(sans = pdfFonts()[["Helvetica"]])
postscriptFonts(sans = postscriptFonts()[["Helvetica"]])

其余代码不需要改动——你已经到处显式写了 fontfamily = "sans"，映射修好后就全部走 Helvetica 了。

还需要什么？

`pdfFonts()` 映射无效，说明 macOS 中文 locale 下 R 的 PDF 设备字体解析绕过了这层映射。有两个可靠方案：

---

**方案 A（推荐）：`showtext` 接管字体渲染**

在所有代码最前面加：

r
library(showtext)
showtext_auto()


如果未安装：`install.packages("showtext")`

---

**方案 B（不装新包）：全部替换为 `"Helvetica"`**

`"sans"` 在 macOS 中文 locale 会被系统解析为 `"Heiti SC"`，而 `"Helvetica"` 是 PDF 设备内建字体，不经过系统映射。把代码中所有 `fontfamily = "sans"` 和 `base_family = "sans"` 和 `family = "sans"` 全替换为 `"Helvetica"`：

r
par(family = "Helvetica")

# ggplot
theme_bw(base_family = "Helvetica")
family = "Helvetica" # geom_text_repel

# ComplexHeatmap gpar
gpar(fontfamily = "Helvetica")


另外 `HeatmapAnnotation` 里的 `gp = gpar(fontfamily = "sans")` 控制的是注释色块而非文字，可以直接删掉：

r
ha_deg <- HeatmapAnnotation(
Group = annotation_col$Group,
col = list(Group = group_colors),
show_annotation_name = FALSE,
annotation_legend_param = list(Group = list(title = NULL))
# 移除 gp = gpar(fontfamily = "sans")
)

确认了：`"Heiti SC"` 不是你代码传的，是 macOS 中文 locale 下 R 的系统默认字体。任何 `fontfamily` 参数替换都无法解决，因为 ComplexHeatmap 内部 `pdf(file = NULL)` 时 grid 继承的是系统默认。

**两个可行方案：**

---

**方案 1（零依赖，临时改 locale）：**

在所有 ComplexHeatmap 代码之前执行：

r
Sys.setlocale("LC_ALL", "C")


这会让 R 不再把默认字体映射到 Heiti SC。中文 `cat()` 输出可能受影响，但绘图不会。跑完后可恢复：

r
Sys.setlocale("LC_ALL", "zh_CN.UTF-8")


---

**方案 2（一劳永逸，装 showtext）：**

r
install.packages("showtext")
library(showtext)
showtext_auto()


加在脚本最开头，之后所有字体问题永久消失。

# =========================
# 下载 GPL23191 注释并映射 ID → SYMBOL
# =========================
library(GEOquery)
library(org.Hs.eg.db)
library(AnnotationDbi)

# 1. 下载 GPL 注释
gpl <- getGEO("GPL23191", destdir = tempdir())
ann <- Table(gpl)

cat("GPL23191 行数:", nrow(ann), "\n")
cat("列名:", paste(colnames(ann), collapse = ", "), "\n")
cat("ORF 非空比例:", mean(ann$ORF != "", na.rm = TRUE), "\n")

# 2. 清理 GB_ACC（去掉版本号 .1 .2 等）
ann$GB_ACC_clean <- sub("\\.[0-9]+$", "", ann$GB_ACC)

# 3. GenBank Accession → Gene Symbol（通过 org.Hs.eg.db）
# GenBank accession 映射到 Entrez ID
acc2eg <- select(org.Hs.eg.db,
keys = unique(na.omit(ann$GB_ACC_clean)),
keytype = "ACCNUM",
columns = c("ACCNUM", "SYMBOL")
)
acc2eg <- acc2eg[!is.na(acc2eg$SYMBOL), ]
acc2eg <- acc2eg[!duplicated(acc2eg$ACCNUM), ]

cat("成功映射的 accession 数:", nrow(acc2eg), "\n")

# 4. 合并
ann$SYMBOL <- acc2eg$SYMBOL[match(ann$GB_ACC_clean, acc2eg$ACCNUM)]

# 5. 如果 ORF 列有值，优先使用 ORF
has_orf <- !is.na(ann$ORF) & ann$ORF != ""
if (any(has_orf)) {
ann$SYMBOL[has_orf] <- ann$ORF[has_orf]
}

# 6. 输出统计
cat("最终有 SYMBOL 的探针数:", sum(!is.na(ann$SYMBOL)), "/", nrow(ann), "\n")

# 7. 生成 GPL 文件（ID + SYMBOL）
gpl_out <- data.frame(
ID = ann$ID,
SYMBOL = ann$SYMBOL,
stringsAsFactors = FALSE
)

# 去掉没有 SYMBOL 的行（可选，看你需求）
# gpl_out <- gpl_out[!is.na(gpl_out$SYMBOL), ]

write.csv(gpl_out, file = "GPL23191_ID_SYMBOL.csv", row.names = FALSE)
