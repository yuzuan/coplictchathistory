先把问题收敛一下：你现在的合并结果“乱七八糟”的根因多半来自 merge/Reduce 的用法与列名处理不当，我会先指出具体原因，然后给你一段更稳健的整合代码和快速自检步骤。

## 问题诊断（为什么会乱）

- 重复使用 merge(by="Row.names") 会不断创造临时列 Row.names，过程中行顺序被打乱，且 rownames 丢失、再赋值，容易错位。
- 每个文件经 summarise(across(where(is.numeric), mean)) 会保留多个数值列（如果原文件有多列数值），但你最后用 colnames(expr) <- names(data_list) 只按样本数重命名，列数和样本数不一致就会错配。
- 多文件中同名列在 merge 后可能产生 .x/.y 后缀或被覆盖，进一步混乱。
- 合并为全外连接后会有 NA，但没有处理和检查；列顺序也未明确对齐到样本名顺序。

核心目标是：每个文件最终变成“gene_id + 该样本唯一数值列”，然后按 gene_id 全外连接；最后把 gene_id 设为行名，列名严格对应样本名。

## 稳健的合并方案（可直接运行）

假设每个 CSV 至少有一列数值；若有多列数值，我们对这些数值取行均值作为该样本单一指标（你也可以改成选择指定列，比如 counts/TPM）。

# 读取并处理单个文件：聚合到 gene_id 唯一，数值列取均值 -> 折叠成单列，列名=样本名
read_one <- function(f, sample) {
  df <- readr::read_csv(f, show_col_types = FALSE)

  if (!"gene_id" %in% names(df)) {
    stop("文件缺少 gene_id 列: ", basename(f))
  }

  df <- df %>%
    mutate(gene_id = as.character(gene_id)) %>%
    group_by(gene_id) %>%
    summarise(across(where(is.numeric), ~ mean(.x, na.rm = TRUE)), .groups = "drop")

  num_df <- df %>% select(where(is.numeric))
  if (ncol(num_df) == 0) {
    stop("文件没有数值列: ", basename(f))
  }

  # 若有多列数值，取行均值；只有一列则直接用
  value <- if (ncol(num_df) == 1) num_df[[1]] else rowMeans(as.matrix(num_df), na.rm = TRUE)

  tibble(gene_id = df$gene_id, !!sample := value)
}

read_one_dt <- function(f, sample) {
  dt <- fread(f)
  stopifnot("gene_id" %in% names(dt))
  # 聚合到 gene_id 唯一，数值列取均值
  num_cols <- names(dt)[sapply(dt, is.numeric)]

}
