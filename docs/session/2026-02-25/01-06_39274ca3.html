### 2. K-M 曲线 legend.labs = levels() 返回 NULL —— 图表崩溃

中，risk_group 是 ifelse() 生成的**字符向量**，不是 factor。levels() 对字符向量返回 NULL，传给 ggsurvplot 的 legend.labs = NULL 会导致图例错误或报错。

### 3. genecox() 函数硬编码依赖全局变量 train_pd —— 隐性耦合

中 genecox() 直接引用全局的 train_pd：

genecox = function(gene){
  train_pd$gene = gene  # 依赖全局变量 train_pd
  m = coxph(Surv(time, event) ~ gene, data = train_pd)

如果在运行前全局环境中残留旧的 train_pd（比如上一次运行的），函数会**静默使用错误数据**，不会报任何错。

---

### 4. timeROC 时间点硬编码为 c(1,3,5,7) —— 可能超出数据范围

硬编码了 time_points <- c(1, 3, 5, 7)，没有检查数据的最大随访时间。如果某些数据集最大随访时间不足7年，timeROC 在 t=7 的计算会因事件数不足产生不可靠结果或报错。（_完整 版已修复此问题）

---

### 5. DCA 分析方法学上完全错误 —— 最严重的统计谬误

的 DCA 实现有三个致命错误：

- **忽略删失**：用 event_at_time = ifelse(time <= tp & event == 1, 1, 0) 将生存数据简单二值化，在 time > tp 但 event == 0（删失）的样本被错误地标记为"未发生事件"
- **任意 sigmoid 转换**：用 1/(1+exp(-risk_scaled)) 将风险评分强行转换为"概率"，这个概率没有任何统计学意义
- **误用 decision_curve()**：将上述伪概率作为预测变量，用逻辑回归公式 event_at_time ~ clinical_prob + risk_prob 而非生存分析公式，结果完全没有意义

正确做法应使用 rmda::decision_curve() 配合 Surv(time, event) ~ ... 公式，让函数内部正确处理删失数据。

---

### 6. 诺莫图使用离散化的 risk_group 而非连续 risk_score

将连续的 risk_score 按三分位数切成"低/中/高"三个分类变量再放入诺莫图。这样做：
- **丢失大量预测信息**
- 诺莫图对每个病人只给出3种可能的分数，失去了个体化预测的意义
- 标准做法应直接用连续的 risk_score

在 glmnet 的交叉验证（cv.glmnet）中，会对一系列 lambda 值计算交叉验证误差（deviance），产生两个常用的 lambda：

- **lambda.min**：使交叉验证误差**最小**的 lambda。模型复杂度较高，选出的变量较多。
- **lambda.1se**：在 lambda.min 的误差基础上，**加一个标准误（1 standard error）**范围内，最大的那个 lambda。

直观理解：

CV误差
  |

  |     ·        ·  ·  ·  ·
  |   ·                       ·
  |  ·  ← lambda.min（误差最低点）
  |  |--|  ← 1 SE 范围
  |        ↑ lambda.1se（在误差只差一个标准误的范围内，选最简单的模型）

         lambda →（越大 = 惩罚越强 = 变量越少）
