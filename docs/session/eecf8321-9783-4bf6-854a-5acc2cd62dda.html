## 核心问题总结

1. **Cl8/10/11 不应标为 TPC**：它们几乎全来自瘤旁（Tumor < 5%），marker 是成熟神经元（SYT1, RBFOX1, MEG3），应标为 **Neuron**
2. **Cl1 不是 MES**：缺乏 CHI3L1/CD44/VIM/ANXA1，更接近 **AC-like tumor**
3. **Cl3/17 应区分 Microglia vs Macro**：Cl0 是经典巨噬，Cl3/17 的 DOCK/ELMO/MEF2A/PLXDC2 特征更像小胶质细胞
4. **Cl4 核糖体高表达需审慎**：可能是低质量细胞或翻译活跃的巨噬
5. **Cl12/19 是数据集特有的肿瘤亚克隆**（MDM2 扩增），不宜归入 OPC/Pericyte

这是 scanpy 版本的兼容性 bug，categories_order 传入子集时 DotPlot 内部缺少 group_extra_size 属性。改为先子集 adata 再绘图（用 view 避免内存问题）：

修改已生效。问题原因：当前 scanpy 版本的 sc.pl.dotplot 在使用 categories_order 参数传入 **类别子集** 时，内部 DotPlot 对象缺少 group_extra_size 属性，触发 AttributeError。

**修复方式**：去掉 categories_order 参数，改为先用 adata[adata.obs['leiden'].isin(target_cls)] 创建 view 子集，再传入 dotplot。view 不占额外内存，也不会崩 kernel。
