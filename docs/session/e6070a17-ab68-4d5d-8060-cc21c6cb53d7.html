## CRITICAL — 数学/方法论错误

### 1. PCA 降维使用了

（未中心化）

svd = TruncatedSVD(n_components=n_components, random_state=0)

**问题**：

**不做中心化**，它等价于 LSA（潜在语义分析），不是 PCA。真正的 PCA 需要先减去每个基因的均值。未中心化时，**第一主成分捕获的是均值信号而非最大方差方向**，导致所有下游分析（Harmony、UMAP、Leiden 聚类）建立在错误的降维基础上。

**对照 scanpy 标准流程**：

内部会自动中心化数据。

**修复**：替换为

或手动中心化后再用 SVD。

---

### 2. 缺少 scaling（基因标准化）步骤

scanpy 标准工作流是：normalize → log1p → HVG → **scale** → PCA

# scanpy skill 示范的标准流程：
sc.pp.scale(adata, max_value=10)  # ← 这一步被完全跳过了

**问题**：没有 z-score 标准化意味着高表达基因（如核糖体基因）会以绝对表达量碾压低表达基因的信号，**造成 PCA 被少数高表达基因主导**。
